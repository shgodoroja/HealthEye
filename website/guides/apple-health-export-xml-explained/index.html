Apple Health export.xml explained: what's actually inside

The Apple Health export delivers an XML-heavy archive. Here is a plain-language walkthrough of what the file contains, what matters for coaching, and where people get stuck.

Published April 2026 · For coaches and analysts

Quick answer

The Apple Health export produces an archive that contains export.xml (the main file with records and workouts), export_cda.xml (a clinical-format variant), and an electrocardiograms folder when present. For coaching, export.xml is where nearly all the useful data lives.

File structure

  • export.xml — all records, workouts, activity summaries, metadata
  • export_cda.xml — Clinical Document Architecture format (not commonly used by coaches)
  • electrocardiograms/ — ECG CSV files when the client has Apple Watch ECG recordings
  • workout-routes/ — GPX files for outdoor workouts when available

Record types that matter

Within export.xml, each <Record> element has a type attribute. The most important for coaching:

  • HKQuantityTypeIdentifierStepCount — step count
  • HKQuantityTypeIdentifierHeartRate — heart rate samples
  • HKQuantityTypeIdentifierRestingHeartRate — daily resting heart rate
  • HKQuantityTypeIdentifierHeartRateVariabilitySDNN — HRV (SDNN)
  • HKCategoryTypeIdentifierSleepAnalysis — sleep stages and totals
  • HKWorkoutActivityType* — workout sessions with duration and energy

Important attributes

  • sourceName — which device or app wrote the record (Apple Watch vs iPhone vs third-party)
  • unit — the value's unit, which can vary (bpm, ms, min, kcal)
  • startDate / endDate — sample window in local timezone
  • value — the measurement

Common pitfalls

  • File size. Multi-year exports can exceed 500 MB. Naive parsers run out of memory.
  • Duplicate records. Both iPhone and Apple Watch may log the same metric. Dedupe by sourceName priority before aggregating.
  • Missing days. Treat missing days as missing, never as zero. This is the single biggest analysis error.
  • Timezone drift. Travel weeks will have samples in multiple timezones. Group by local day.
  • Sleep model changes. Apple's sleep stage schema changed; older records only have in-bed vs asleep.
