Use Case

Weekly client recovery report in under five minutes per client

How an independent performance coach runs a repeatable recovery check-in using HealthEye on Mac — from Apple Health export to client-ready PDF.

The weekly loop

  1. Collect. Each client exports Apple Health from iPhone on Sunday evening and shares the archive.
  2. Import. The coach imports on Mac using guided import. Metric completeness is shown per client and week.
  3. Triage. The dashboard ranks clients by attention score so the coach starts with whoever needs the most attention.
  4. Review. For each flagged client, the coach inspects sleep, HRV, resting heart rate, workouts, and steps against the 28-day baseline.
  5. Report. One click produces a PDF with trend deltas, alert summary, and a suggested client message.
  6. Send. The coach pastes the suggested message into the usual client channel and attaches the PDF.

What changes week over week

  • Recovery composite shift versus baseline (sleep + HRV + resting HR)
  • Workout minutes delta with intensity context
  • Step count drop or streak extension
  • New alerts firing this week (AR-001 to AR-004) with severity
  • Completeness penalty if data coverage was thin

Why this workflow holds up at 30–100 clients

The attention score ranking means the coach does not read every client equally. Low-score clients get the short version; high-score clients get the deep review. Score changes are always traceable to metric changes, so when a client asks why the report looks different from last week, the answer is not guesswork.
